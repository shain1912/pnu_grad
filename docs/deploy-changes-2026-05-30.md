# 배포본 → dev 변경 내역 (2026-05-30)

**비교 기준**: 현재 `pnug.kodekorea.kr`에 배포된 빌드 ↔ 로컬 dev(`frontend/public/admission-v*.html`)
**비교 방법**: 라이브에서 받은 빌드 HTML과 로컬 소스를 파일 단위 diff (텍스트 추출 + raw HTML 라인 대조)
**전체 규모**: raw HTML 기준 v1 `+3/−5`, v2 `+8/−10`, v3 `+9/−10` 라인 — **변경은 모두 콘텐츠/카피이며 CSS·레이아웃·디자인·애니메이션은 변경 없음.**

요약: ① 장학금 규정 정확성 보강(v2·v3), ② 신청 모달의 허위 안내("이메일 발송") 정정(3종 공통), ③ 실제 파일 없는 가짜 "다운로드" 제거(3종 공통).

## 1. 3개 시안 공통 변경 (v1 · v2 · v3)

### 1-1. 신청 모달 인트로 카피 — 허위 안내 정정
백엔드(`surveys.js`)는 응답을 SQLite 저장 + Google Sheets 미러만 하고 **이메일을 보내지 않는다.** 기존 문구는 실제 동작과 불일치(허위).

```diff
- 아래 정보를 입력하시면 소속 단과대학 행정실 안내와 함께 지원서 양식이 이메일로 발송됩니다.
+ 아래 정보를 입력하시면 사전 신청이 접수됩니다. 정식 지원서 양식은 소속 단과대학 홈페이지에서 받아 행정실에 제출해 주세요.
```

### 1-2. "학년별 수료학점 기준표" 콜아웃 — 가짜 다운로드 제거
실제 XLSX 파일이 없어 버튼이 가짜 토스트("자료실에서 …다운로드 시작")만 띄웠음. 버튼 제거 + 안내문으로 전환(점선/보더 박스 레이아웃 유지).

```diff
- 학과(부)별 수료학점 및 학기별 이수 기준을 확인하세요. (XLSX)
- [⬇ 다운로드] (버튼)
+ 학과(부)별 수료학점 및 학기별 이수 기준은 소속 단과대학 홈페이지·학과 사무실에서 확인하세요.
```

### 1-3. 최종 CTA — "지원서 다운로드" 버튼 제거
실제 양식 파일 없음(공식 출처는 단과대학 홈페이지). 버튼 제거, 주 CTA `지금 신청하기`만 유지(정렬 그대로).

```diff
  [지금 신청하기 →] (버튼, 유지)
- [지원서 다운로드] (버튼, 제거)
```

## 6. 라이브 반영(재배포) 방법
이 변경은 로컬 `frontend/public/`에만 있음. `pnug.kodekorea.kr`에 반영하려면 프론트 빌드 후 prod 재시작:

```cmd
:: 루트에서
start-prod.bat        :: = frontend 빌드(public 복사) + backend(Express) 기동
:: Caddy는 기존대로 (start-caddy.bat)
```
정적 `public/` 파일이라 빌드는 사실상 복사이며, OAuth/DNS/인증서 등 인프라 설정은 건드릴 필요 없음.
