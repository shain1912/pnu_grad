# ADR 0005 — 응답 데이터 최소화 + 3지망(학과+전공) 모델

- 상태: Accepted
- 날짜: 2026-06-01

## 컨텍스트

클라이언트(교수) 피드백:
- **개인정보 수집에 부담**이 있다. `@pusan.ac.kr` 이메일만 있으면 내부 시스템에서 신원을 식별할 수 있으므로 학번·이름 등은 꼭 필요하지 않다.
- 신청의 의미를 "희망 진학 **학과 3지망**"으로 확장한다. 학생은 학과 디렉터리(계열/BK21/키워드 필터)에서 학과를 고르고 그 학과의 전공 중 선택한다.
- 평점·학점·학기는 **별도 자격 확인 페이지**에서 자가 입력해 자격 여부만 확인한다(신청 저장과 무관).

기존 모델(ADR 0001 시트 스키마, `init-db.js` survey Q1~Q6)은 학번·소속학과·이수학기·트랙·동의를 수집해 SQLite + Google Sheets에 저장했다 — PII가 외부 클라우드로 나가는 구조.

## 결정

**응답에 저장하는 항목을 최소화한다: 검증된 이메일 + 희망 트랙 + 3지망(각 학과+전공).**

- 이메일: **OAuth 유지**(ADR 0002의 신청자 OAuth + `@pusan.ac.kr` 도메인 게이트). 검증된 이메일 확보 + 1인 1응답(`UNIQUE(survey_id, user_id)`) 유지. **이름·사진은 저장하지 않음.**
- 희망 트랙: ①학·석사 / ②학·석박사통합 / ③전환 (제도 유형 — PII 아님).
- 3지망: 우선순위 1~3, 각 (학과, 전공). 최소 1, 최대 3. 선택지는 일반대학원 모집(○) 학과(모집요강 기준).
- **학번·이름·평점·이수학기는 수집·저장하지 않는다.** 평점·학점·학기는 자격 확인 페이지에서 자가 입력만 하고 저장하지 않는다.
- 시트/SQLite/모달 폼/admin 통계를 위 스키마로 변경(시트 컬럼: email·트랙·1·2·3지망).

## 결과

긍정:
- PII 노출 최소화 → 학사 정보보호 컴플라이언스 리스크 대폭 감소(이메일만 외부 저장).
- 신청 UX 단순화: 로그인 후 트랙+3지망 선택만, 타이핑 PII 0.
- 행정은 이메일→내부 식별 + 3지망/트랙으로 처리 가능.

부정:
- admin이 시트에서 이름/학번을 바로 못 봄 → 이메일로 내부 조회 필요.
- ADR 0001의 시트 컬럼 스키마 일부를 **supersede**. `init-db.js` survey 정의, 세 시안의 신청 모달, admin 대시보드 통계(이수학기 분포→지망/계열 분포)를 전면 수정해야 함.
- 기존 응답 데이터(있다면)와 스키마 불일치 → 마이그레이션 또는 초기화 필요.

## 검토한 대안

- **현행 PII(학번·학과·이수학기) 유지 + 3지망만 추가** — 클라이언트의 개인정보 최소화 요구에 정면 배치. 기각.
- **OAuth 제거하고 이메일 직접 입력** — 로그인 마찰은 없으나 이메일 검증·1인 1응답·도메인 게이트가 약화(오타·도용). 기각([[ADR 0002]] 유지).
- **자격 확인 결과(평점·학점)를 응답에 함께 저장** — 가장 민감한 정보를 저장하게 되어 최소화 취지에 정면 배치. 자격 확인은 클라이언트 사이드 자가확인으로만. 기각.
