# ADR 0004 — 자료실·공지 게시판 없음, 양식은 단과대학 홈페이지로 위임

- 상태: Accepted
- 날짜: 2026-05-29

## 컨텍스트

원본 안내 페이지(GitHub `grad_s3_3_uglink_guide.html`)에는 "공지사항·자료실(NOTICE BOARD)" 섹션과 지원서·리플렛·포스터·수료학점 기준표의 다운로드/파일 업로드 UI가 있었다. 그러나 그것들은 **정적 데모**였다 — `fakeDownload()`가 토스트만 띄우고 실제 파일·백엔드가 없었다.

pnug를 이식하면서 시안(`admission-v1/v2/v3.html`)에도 같은 다운로드 버튼이 "자료실에서 …다운로드를 시작합니다"라는 가짜 토스트로 남아 있었고, 신청 모달은 "지원서 양식이 이메일로 발송됩니다"라고 안내했다. 둘 다 실제로는 동작하지 않는다(백엔드에 파일 호스팅·게시판·메일 발송 라우트 없음 — `server.js`는 `/auth`·`/api/surveys`·`/api/admin`만).

## 결정

**시안 페이지의 범위는 (1) 제도 안내 + (2) 사전 신청(응답) 수집 으로 한정한다. 공지사항·자료실 게시판, 파일 업로드/호스팅은 범위 외.**

- 정식 **지원서 양식·모집요강·학년별 수료학점 기준표** 등 문서의 공식 출처는 **소속 단과대학 홈페이지/학과 사무실**이다. 본 시스템은 이 파일들을 호스팅하거나 다운로드를 제공하지 않는다.
- 실제 파일이 없는 "다운로드" 버튼과 가짜 토스트는 **제거**한다. 안내가 필요한 자리에는 "소속 단과대학 홈페이지에서 확인" 문구로 대체한다.
- 신청 모달 카피는 미구현 기능("이메일로 발송")을 빼고 실제 동작("사전 신청 접수")과 단과대학 홈페이지 안내로 정정한다. → 사전 신청(응답)과 정식 지원서는 구분된다([`CONTEXT.md`](../../CONTEXT.md) 참조).

## 결과

긍정:
- 허위 다운로드·이메일 안내 제거 → 사용자에게 정직한 UI
- 백엔드에 파일 저장·게시판·메일 라우트 불필요 → 스코프·유지보수·PII 노출 축소
- 시안 디자인은 그대로 — 버튼 1개 제거/문구 교체 수준의 최소 변경

부정:
- 학생이 사이트에서 양식을 바로 받지 못하고 단과대학 홈페이지로 이동해야 함
- 원본 안내 대비 자료실·공지 기능 부재(의도적). 미래 독자가 "왜 빠졌나" 물을 수 있어 본 ADR로 기록

## 검토한 대안

- **게시판 + 파일 호스팅 백엔드 구축** — 업로드/목록 라우트, 파일 스토리지, (PII 포함 가능) 첨부 관리가 필요. 트래픽·운영 대비 과하고 정보보호 위험 증가. 기각.
- **원본처럼 가짜 데모 유지** — 클릭해도 아무 일도 안 일어나는 허위 UX. 기각.
- **단과대학 페이지로 직접 링크하는 버튼** — 단과대학이 여러 곳이라 단일 URL이 없음. 버튼 대신 "소속 단과대학 홈페이지에서 확인" 안내 문구로 대체.

향후 학사가 단일 자료실(공식 파일 호스팅)을 요구하면 별도 ADR로 이 결정을 supersede 한다.
